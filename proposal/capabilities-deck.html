R1 — Reports remediation

Fix it completely.
Not partially.

What we found (pre-engagement audit)
  • Quarterly and monthly-trends endpoints missing warehouse, category, and status filters
  • Inconsistent API patterns vs. all other endpoints
  • Options API remnants mixed with Composition API — regression risk on every edit
  • i18n gaps across report labels and date formats
  • Console errors on filter state changes
Our approach
  • Audit-first: enumerate all defects before writing a line of code
  • Fix in order of operational impact
  • Each fix ships with a corresponding test (R3 and R1 are linked)
  • Complete Composition API migration as part of remediation
  • Sign-off checklist shared with Meridian IT before closing R1

to one screen.

Input

Logic

What the system calculates

Output

What the operator sees

R3 — Automated browser testing

Tests first.
So IT says yes.

Coverage plan — 5 critical flows
  • Inventory view — load and filter by warehouse + category
  • Orders view — filter by status and time period
  • Reports view — quarterly and monthly trends with filters
  • Restocking view — budget input → recommendations generated
  • Dashboard summary — KPI accuracy with combined filters
Why Playwright
  • Industry standard for Vue/JS applications
  • Already configured in the project's MCP setup
  • Tests run against localhost — no external dependencies
  • Readable, maintainable by Meridian IT after handoff
  • Delivered in Phase 1 — unblocks all subsequent work
Delivery order matters: R3 ships in Phase 1, before R1 fixes and R2 build. This is what unlocks IT approval for everything that follows.
